# Правила проекта

## Рабочий процесс

- **Ревью перед коммитом.** Не выполнять `git commit` / `git push` автоматически.
  Сначала внести изменения, прогнать проверки (typecheck/тесты), показать сводку
  и дать пользователю отревьюить дифф. Коммитить и пушить — только после явного
  одобрения пользователя.

## Контекст

- Архитектура, синхронизация и фичи описаны в `DOCS.md`; запуск — `README.md`;
  деплой и ловушки данных — `DEPLOY.md`.
